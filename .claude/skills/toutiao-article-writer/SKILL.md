# 头条号文章写作技能

本 skill 负责将搜索结果转化为头条号文章，执行完整的七步写稿流程。

## 核心原则

1. **点击优先**：标题要有悬念/情绪/代入感
2. **活人感**：口语化表达，避免 AI 味
3. **信息密度**：不要注水，每句话都有价值
4. **情绪节奏**：开头炸 → 中间魔幻 → 结尾有观点
5. **短句短段**：适合手机阅读，段落不超过 3 行

## 输入要求

从 `work/YYYY-MM-DD/3_search/{话题名}/` 目录读取 10 轮搜索结果。

## 输出要求

到 `work/YYYY-MM-DD/4_output/{话题名}/` 目录产出 7 个文件：

1. `01_thinking.md` - 写作优化思考
2. `02_outline.md` - 文章大纲
3. `03_draft.md` - 初稿
4. `04_review_advice1.md` - 审稿建议1
5. `05_draft1.md` - 修改稿1
6. `06_review_advice2.md` - 审稿建议2
7. `07_final.html` - 最终 HTML 成品

## 七步写稿流程

### 第 1 步：01_thinking.md - 写作优化思考

分析话题、规划结构、确定标题方向。

```markdown
# 写作优化思考

## 话题分析
- 话题：{话题名}
- 核心冲突：{是什么让人想点开看？}
- 情绪触发点：{好奇/愤怒/同情/惊讶}

## 读者痛点
- {读者为什么关心？}
- {这事儿跟他们有什么关系？}

## 标题方向（选 3 个）
1. {标题1} - {为什么好}
2. {标题2} - {为什么好}
3. {标题3} - {为什么好}

## 结构规划
- 开头：{直接甩最炸的点}
- 展开：{逐步展开故事}
- 转折：{有什么反转/意外}
- 收尾：{观点或警示}

## 情绪节奏
- 开头：{什么情绪}
- 中间：{什么情绪}
- 结尾：{什么情绪}
```

### 第 2 步：02_outline.md - 文章大纲

详细的写作大纲，包含每个部分的要点。

```markdown
# 文章大纲

## 一、开头（钩子）
- {场景/事实}
- {时间/地点/人物}
- {悬念设置}

## 二、展开
- {要点1}
- {要点2}
- {要点3}

## 三、转折
- {反转/意外}
- {当事人回应}

## 四、收尾
- {观点/警示}
- {情绪落点}

## 预计字数：{800-1200}字
```

### 第 3 步：03_draft.md - 初稿

直接开始写，不要太纠结，先把内容写出来。

**写作要求**：
- 短句为主，10-20 字一句
- 段落不超过 3 行
- 用数字、对话、场景
- 避免"这说明了""值得注意的是"等 AI 味
- 标题要炸，有数字/反转/悬念

### 第 4 步：04_review_advice1.md - 审稿建议1

以编辑视角审视初稿，提出改进建议。

```markdown
# 审稿建议1

## 整体评价
- {✅ 做得好的地方}
- {⚠️ 需要改进的地方}

## 修改建议
1. 【标题】{建议}
2. 【结构】{建议}
3. 【细节】{建议}
4. 【去AI味】{建议}

## 重点修改
- {具体指出哪几段需要改}
```

### 第 5 步：05_draft1.md - 修改稿1

根据审稿建议1进行修改。

### 第 6 步：06_review_advice2.md - 审稿建议2

第二轮审视，重点看活人感和信息密度。

```markdown
# 审稿建议2

## 整体评价
- {✅ 改进后的优点}
- {最后需要打磨的点}

## 建议采用
- 标题：{最终标题}
- 结尾：{最终结尾}
```

### 第 7 步：07_final.html - 最终 HTML 成品

使用头条号专用 HTML 模板（见 `references/html-template.md`）。

见 `references/html-template.md`。

- 引用框/强调框用不同背景色

## 标题技巧

### 好标题的特征

1. **有数字**：30年、20岁、240万、两天两夜
2. **有反转**：医生中毒进自己科室、树冒烟全是虫
3. **有悬念**：到底咋回事？真的假的？
4. **有情绪**：炸死、中毒、昏迷、全家

### 标题公式

```
{人物/主体} + {意外事件} + {数字/细节}
例：云南医生吃菌子中毒住进自己医院：全家连续三年中招
```

```
{反常识现象} + {真相}
例：远看树冒烟近看全是虫，走近看头皮发麻
```

## 去AI味检查清单

- [ ] 没有"这说明了""值得注意的是"
- [ ] 没有"首先/其次/最后"
- [ ] 没有"综上所述""总而言之"
- [ ] 段落长度有变化，不都一样长
- [ ] 有对话、场景、数字
- [ ] 结尾有观点，不是道德说教
