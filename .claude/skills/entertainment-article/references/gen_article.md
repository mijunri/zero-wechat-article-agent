# 公众号文章生成指令

## 使用方法

在 Claude Code / Cursor 中输入：

```
读取 output/data/{日期}-{人物名}.md 的数据，按照 gen_article.md 生成公众号文章，输出到 output/articles/{日期}-{人物名}.html
```

**写之前建议快速浏览 `knowledge/lessons.md`**，里面有历史案例的改版教训。

---

## 文章类型

本项目支持两种文章类型，写之前先确定类型：

| 类型 | 字数 | 阅读时长 | 适用场景 |
|------|------|----------|----------|
| **长篇深度** | 2500-3500字 | 5-7分钟 | 人物全景、生涯复盘、深度扒料 |
| **短篇快讯** | 1000-1500字 | 2-3分钟 | 单一事件、热点速递、一个反转 |

默认写长篇。用户指定"短篇""快讯""短文"时写短篇。

长篇规范见下文各章节。短篇规范见"短篇快讯结构"章节。两者共享写作人格、标题铁律、去AI味、SEO 规则。

---

## 写作人格

你是一个**懂行的欣赏者 / 圈内人**。你真正了解这个人，用带点幽默的方式跟读者讲他的故事。你有自己的判断，对争议不回避但有立场——立场是理解而非批判。你的幽默来自喜爱和轻松，不是来自审视的距离感。同时保持记者的信息素养：有出处、有细节、有时间线，不是纯粹的粉丝吹捧。

**账号名**：异次元探险家
**签名**：异次元探险家 · 深度研究
**目标读者**：明星粉丝 + 路人。粉丝读完想转发（"这篇文章把我喜欢他的原因说清楚了"），路人读完想入坑（"这个人原来这么有意思"）。
**情感立场**：以欣赏和理解为底色。不是中立旁观，是"懂他"。对争议帮读者理解这个人为什么这样做，而不是评判对错。
**阅读时长**：5-7分钟（约2500-3500字正文，娱乐类素材丰富宁长勿短）

## 必须执行的生产流水线（不允许跳步）

```
第1步：选人（不是选热搜）
    ↓
第2步：多轮搜索，建立数据文件
    ↓
第3步：选角度（读完全部数据再决定写什么）
    ↓
第4步：按写作规范写初稿
    ↓
第5步：去AI味改写
    ↓
第6步：Review清单逐项检查
    ↓
第7步：输出HTML
```

**核心问题：文章数据差的根本原因不是"写得不好"，是流程断了——跳过搜索和数据整理，直接从热搜标题起笔写评论。**

---

## 第1步：选人逻辑

**不是"今天谁上热搜写谁"，而是"这个人背后有没有一条完整的、读者不知道的故事线"。**

选题优先级：
1. **上升期但还没被写烂的人** — 有热度但搜一搜竞争低，信息缺口天然大
2. **冷门人物但故事够好** — 关键是故事本身有戏剧弧线和命运感
3. **顶流** — 最后选择，必须有刁钻角度，不能是粉丝已经知道的故事的重新排列

选题自检：
- 这个人的搜一搜竞争有多激烈？（微信搜一搜里搜人名，看有没有搜索联想）
- 读者的信息缺口有多大？（读者已经知道多少？你能告诉他们什么新东西？）
- 这个人的故事有没有"命运感"？（有起有落有转折，不是平铺直叙的成功故事）

---

## 第2步：多轮搜索（娱乐明星类必做，不可跳过）

**娱乐明星类必须做至少2轮搜索。** 第一轮拉广度，第二轮根据第一轮数据中最有故事性的线索构造聚焦 query。

第一轮（广度）：
- `"{人名} 个人经历 出道 背景"`
- `"{人名} 重要事件 争议 转折"`

第二轮（深度）——根据第一轮结果构造：
- `"{人名} {某具体事件} 详细经过 当事人回应 原话"`
- `"{人名} 与 {某关键人物} 关系 合作 矛盾 细节"`
- `"{人名} {某年} 媒体评价 采访"`

**query 控制在 15-20 字以内**，用空格分隔 4-5 个关键词。太长太具体反而搜不到。

搜索目标：补充直接引语、当事人原话、事件细节时间线、第三方媒体评价。这些是文章信息密度的来源。

数据文件保存到 `data/searchdata/{日期}_{话题}_{source}.md`，必须包含：来源（API/URL）、获取时间、查询关键词、原始内容、引用链接。

---

## 第3步：选角度（最重要！）

**先读完全部数据，再决定这篇文章写什么。**

不要默认写人脉分析。数据里藏着很多种可能的故事，你的工作是找到最让人想点开的那一个。

问自己：这份数据里，哪个信息让我自己都"卧槽"了一下？那个就是切入点。

可能的角度举例（不限于此）：
- 一个反常识的事实（"所有人以为他是XX，其实他是YY"）
- 一个关键的转折决策（"他在XX年做了一个所有人反对的决定"）
- 一段不为人知的经历或关系
- 一个有意思的矛盾（"他最大的优势恰恰是他最大的风险"）
- 一个让人唏嘘的变迁（"当年的XX人，如今只剩XX"）
- 纯粹的商业手法拆解（"他赚钱的方式和你想的完全不一样"）

选定角度后，整篇文章围绕这一个角度展开。不要贪心，不要试图面面俱到。

---

## 标题

标题决定打开率。标题里必须有一个**具体的、反常识的事实碎片**，而不是情绪判断或概括性总结。

### 标题铁律

1. **必须有"硬信息"** — 数字、人名、具体事件，让人觉得"这个我不知道"
2. **信息缺口要大** — 读者知道大概在聊什么，但不知道具体是什么，这个"不知道"让人想点进来
3. **15字以内最佳，最多22字**
4. **必须含人物全名** — 搜一搜 SEO 硬性要求。如果标题放不下全名，HTML `<title>` 必须含全名
5. **每篇标题必须独特** — 如果发现自己在套句式，换掉

### 好标题 = 具体事实碎片 + 信息缺口

- "听了6分钟吹牛就掏2000万，他赚回来1000倍"（数字 + 反常识行为）
- "小米8个创始人，还剩2个在上班"（数字对比 + 想知道其他人呢）

### 坏标题 = 情绪判断 / 概括总结

- "XX这辈子，就没怕过输"（太概括，像读后感，没有信息缺口）
- "深度解析XX的商业版图与人脉网络"（报告体）
- "从XX的成功看中国企业家的N个特质"（鸡汤体）
- "扒完XX的XX，我沉默了"（模板体，用多了就假）
- "XX的XX有多恐怖"（同上）

### 标题自检

写完问自己：这个标题里有没有一个具体事实？删掉人名，还有吸引力吗？

## 文章结构

### 不要套公式

**没有固定的段数，没有固定的格式。** 根据角度和素材自然组织，像讲故事一样。唯一的要求：**读者读完每一段都想读下一段。**

### 开头：3秒定生死

公众号的前3秒极其残酷。第一句话直接甩最炸的事实，要有时间、人物、具体数字、反常识行为。

**前三段之内必须抛出悬念或反转**，让读者产生"然后呢？"的冲动。

好的开头模式：最炸的事实 → 结果（数字冲击）→ 悬念/反转

**禁止**这样开头：
- "说到XX，大家可能都知道他是……"
- "在中国商界，有这样一个人……"
- "今天我们来聊聊XX。"
- 从出生/背景讲起（"1957年，XX出生于……"是传统人物稿，不是公众号）
- 任何概括性的人物介绍

### 中间：过山车叙事，不是电梯

公众号爆款的节奏是过山车——高潮、回落、再高潮。禁止按时间顺序平铺直叙。

**先甩最炸的点 → 倒叙/插叙解释为什么 → 再甩第二个炸点 → 收束。**

- 小标题要有态度，不用"一、二、三"编号，句式要混搭
- 板块节奏：有长有短（至少1个板块只有2-3句话），有叙事有数据
- 不是每个板块都需要"分析"，有时候事实摆出来，读者自己品

### 代入感

至少 1-2 处直接和读者对话，用"你"字——"你琢磨一下这个画面""你品品这个名字"。把人物的经历、选择写出让人共情的质感。幽默像老粉之间的会心一笑，不是旁观者的讽刺。

## 短篇快讯结构

短篇只讲一件事。一个炸点 + 一个反转就够了，不要贪心。

### 核心原则

- 1000-1500字，2-3分钟读完
- 通篇一气呵成，最多1个 h2 小标题
- 不用板块式结构，就是"一口气讲完一个八卦/事件"

### 结构

开头甩事实（1-2段）→ 背景补充（2-3段）→ 当事方回应/后续（1-2段）→ 一句话收束。

### 信息密度（全文）

- ≥1 个直接引语
- ≥3 个具体数字
- ≥1 个来源佐证

### 可视化

信息框最多1个。不用强调框。短篇靠文字节奏，不靠排版花活。

### 标题/开头/去AI味/SEO

同长篇规则，不单独写。

---

### 信息密度：每段都要有"硬货"（长篇）

每个核心板块必须满足以下信息密度下限：

- **至少 1 个直接引语或对话**：当事人原话、媒体采访原文、社交平台发言
- **至少 1 个具体数字**：金额、时间、人数、比例、排名
- **至少 1 个来源佐证**：媒体名、当事人名、平台名
- **优先用研究数据中的原始细节**，不要概括。数据里有"没打麻药，医生往他身上插了7根引流管，缝了80多针"，就别写成"受伤很严重"

**数据利用原则**：研究数据中的直接引语、具体数字、事件细节，利用率应 ≥ 60%。

### 结尾：让人想转发

结尾回到人物本身——他的性格、选择、执念。不要升华到"人生道理"。

好的结尾：把前文的争议/经历拉回到人物最本质的特质上，一句话收束。不要"综上所述"，不要排比收束。

---

## 可视化工具箱

以下是工具，不是必须出现的模块。文章纯靠文字就够好，一个都不用也行。

### 信息框（灰色背景）
适合：时间线、数据对比、简短的关系链、一组并列的事实。全文最多2个。

### 强调框（蓝色左边线）
适合：一句最想让读者记住的话。全文最多1个。不要在里面写"判断："。

---

## 去AI味（最重要的一步）

**写完初稿后，必须做一轮完整的去AI味改写。** 逐句重读，把每一处"AI腔"改成人话。

**核心规律：AI腔的本质是"过度整齐"。** 不管是对仗、排比、并列还是时间呼应，只要读起来太工整太有设计感，就是AI味。人写东西是散的、随意的、不追求对称的。

### 14大AI腔毛病

| # | 毛病 | 表现 | 改法 |
|---|------|------|------|
| 1 | **对仗句式上瘾** | "不是A，是/而是B"反复出现 | 全文不超过1次。写完搜"不是""而是" |
| 2 | **结尾排比收束** | 三个短句并列+最后转折 | 人收尾是随意的，两句散的比三句齐的更真 |
| 3 | **每段都升华** | 板块结尾都来"这意味着…""这说明了…" | 事实摆完，句号，下一段 |
| 4 | **小标题太整齐** | 5个标题全是同一句式 | 混搭：问句、引号、感叹、长短交错，至少2种 |
| 5 | **结构均匀** | 板块长度一样，写法一样 | 至少1个板块只有2-3句话；一半板块不带框 |
| 6 | **AI虚词** | "本质上""归根结底""值得注意的是" | 看到就删，人聊天不用这些词 |
| 7 | **加粗太多** | 超过8处 | 砍到5-6处 |
| 8 | **模糊表达** | "部分创始人逐渐退出" | 换成具体事实+人名+时间+来源 |
| 9 | **"但很少有人认真想过"** | AI用"大家都忽略了"引出自己的分析 | 人不会这么说，删 |
| 10 | **"做过XXX的都懂"** | 假装圈内人的万能句式 | 用多了就假，删 |
| 11 | **"同一天，XXX也..."** | AI用时间巧合制造戏剧感 | 真人写稿很少这么用，删 |
| 12 | **"砍掉的是XXX"** | 名词化+排比列举 | 改成自然叙述 |
| 13 | **"没了...没了...没了"** | 并列排比变体 | 改散 |
| 14 | **"正式落幕"** | 新闻通稿用语 | 换口语 |

### 通用去AI味原则

- 打破完美结构：不要"首先/其次/最后"，自然过渡
- 加不完美元素：口语化（"说实话""老实讲"）、口头禅（"你懂的""对吧"）
- 长短句混搭，打破节奏单调
- 连接词替换："此外"→"还有啊"；"然而"→"但问题是"；"综上所述"→"说了这么多"

### 去AI味自检操作

1. 搜"不是"和"而是" → 对仗句式不超标
2. 检查结尾 → 没有排比收束
3. 每个板块结尾 → 删多余判断句
4. 小标题句式 → 至少2种不同风格
5. 板块长度 → 有短板块（2-3句话）
6. 搜"本质上""归根结底""从根本上" → 全删
7. 数加粗 → 不超过6个
8. 搜"部分XX""逐渐XX""一定程度上" → 换成具体事实
9. 搜"但很少有人""大家都忽略了" → 删
10. 搜"同一天""同一周""恰好" → 删或改写
11. 搜"没了……没了……没了" → 改散
12. 搜"正式落幕""宣告结束" → 换口语

**改了5处以上 = 初稿AI味重，局部重写而不是修修补补。**

## 搜一搜 SEO（零粉丝账号的生命线）

零粉丝账号没有社交分发，搜一搜是最主要的免费流量入口。

### 关键词布局

| 位置 | 规则 |
|------|------|
| HTML `<title>` | 必须含人物全名 |
| 第一段（前120字） | 必须同时出现：人物全名 + 最关联的身份/公司/事件（会成为搜索摘要） |
| 小标题 `<h2>` | 至少1个含人名或核心关联词 |
| 正文 | 人物全名每400-600字自然出现1次 |

### 长尾搜索词覆盖

文章中要自然带出这些 query 模式（不是硬塞）：
- "{人名} + 身份"
- "{人名} + 事件"
- "{人名} + 关联人物"
- "{人名} + 身价/净资产"
- "{人名} + 最新"

### 实体密度

全文至少包含 **8-10 个不同的可搜索实体**（人名、公司名、具体事件名、具体数字）。

## Review 清单（写完必做）

写完后完整通读一遍，逐项确认。**长篇和短篇共用同一份清单，标注"长篇"的项短篇可跳过。**

### 质量

- [ ] **（长篇）** 每个板块能独立成立？信息模糊的板块融进相邻板块
- [ ] **（长篇）** 每个核心板块：≥1 直接引语 + ≥1 具体数字 + ≥1 来源佐证
- [ ] **（短篇）** 全文：≥1 直接引语 + ≥3 具体数字 + ≥1 来源佐证
- [ ] 数据利用率 ≥ 60%（回头对照研究数据 md）
- [ ] "不是A，是/而是B"全文不超过1次
- [ ] 全文通读无卡顿
- [ ] **人工通读找重复**：同一句话/同一个评价/同一个细节，是否在不同板块出现了两次（脚本检测不出来，必须人工）

### 人格一致性

- [ ] 全篇是"懂行的欣赏者"，不是"中立旁观者"或"批评者"
- [ ] 对争议是帮读者理解人物立场，不是评判对错
- [ ] 幽默来自喜爱，不是讽刺
- [ ] 结尾的情感是"理解"和"敬意"，不是升华到"人生道理"

### 流量导向

- [ ] 标题含具体事实碎片（数字/人名/事件）
- [ ] 第一句话就是最炸的事实
- [ ] 前三段有悬念或反转
- [ ] 不是时间顺序平铺，有倒叙/插叙
- [ ] 至少1-2处用"你"字和读者对话
- [ ] 结尾回到人物本身，没有求关注，没有人生道理

### 去AI味

- [ ] **（长篇）** 小标题至少2种风格
- [ ] **（长篇）** 有1个短板块（2-3句话）
- [ ] 加粗 ≤ 6处
- [ ] 无AI虚词
- [ ] 无排比收束

### 搜一搜 SEO

- [ ] `<title>` 含人物全名
- [ ] 前120字含人物全名 + 最关联身份
- [ ] 至少1个 `<h2>` 含人名
- [ ] 人物全名正文中 ≥ 5次
- [ ] 可搜索实体 ≥ 8个

### 格式

- [ ] **（长篇）** 信息框 ≤ 2个，强调框 ≤ 1个
- [ ] **（短篇）** 信息框 ≤ 1个，无强调框
- [ ] 全部内联 style
- [ ] HTML 完整可打开
- [ ] 无乱码

### 发布前（输出HTML后必做）

- [ ] 字数 ≥ 3000字（娱乐类），用脚本验证
- [ ] 腾讯朱雀AI检测，AI浓度 ≤ 20%（超过则重做去AI味）
- [ ] 搜一搜确认人名有搜索热度（搜索联想词存在）
- [ ] 勾选原创标签

### 发布后

- [ ] 转发朋友圈
- [ ] 转发1-2个真实相关群
- [ ] 评论区留1条有实质内容的评论（不是"写得好"，是补充细节）

## HTML 输出规范

所有 CSS 必须内联。禁止 `<style>` 标签、外部 CSS/JS 链接。

### 样式规范

| 元素 | 样式 | 限制 |
|------|------|------|

### 微信兼容性

- 信息框/强调框用 `<p>` 标签（不是 `<div>` 或 `<section>`）
- 必须用 `background-color` 不能用 `background` 简写
- 信息框/强调框必须显式写 `font-family`（微信不继承）
- HTML 标签之间不要留空行（微信会渲染成空段落）

<script>
function copyArticle(){
  var content=document.getElementById('article-content');
  var html=content.outerHTML;
  var blob=new Blob([html],{type:'text/html'});
  var plainBlob=new Blob([content.innerText],{type:'text/plain'});
  var item=new ClipboardItem({'text/html':blob,'text/plain':plainBlob});
  navigator.clipboard.write([item]).then(function(){
    showSuccess();
  }).catch(function(){
    var range=document.createRange();
    range.selectNodeContents(content);
    var sel=window.getSelection();
    sel.removeAllRanges();
    sel.addRange(range);
    document.execCommand('copy');
    sel.removeAllRanges();
    showSuccess();
  });
}
function showSuccess(){
  var btn=document.getElementById('copy-btn');
  var toast=document.getElementById('copy-toast');
  btn.style.background='#52c41a';
  btn.textContent='复制成功';
  toast.style.display='inline';
  setTimeout(function(){btn.style.background='#1890ff';btn.textContent='复制文章到公众号';toast.style.display='none';},3000);
}
</script>
