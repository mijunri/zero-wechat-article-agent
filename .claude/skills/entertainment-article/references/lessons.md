## 标题案例

### 好标题（已发布，有数据验证）

| 人物 | 标题 | 好在哪 |
|------|------|--------|
| 孙正义 | 听了6分钟吹牛就掏2000万，他赚回来1000倍 | 两个具体数字 + 反常识行为 + 想知道是谁 |
| 黄仁勋 | 英伟达三个创始人，只有皮衣那个赢麻了 | 数字对比 + "皮衣"辨识度 + 想知道其他人呢 |
| 张凌赫 | 张凌赫曾胖到120公斤，差点进了国家电网 | 具体数字 + 反差（偶像剧男主vs120公斤）+ 意外人生岔路 |
| 李荣浩 | 李荣浩19岁断了6根肋骨，躺病床一年写出半张专辑 | 具体数字×3 + 反差（病床vs创作）|
| 肖战 | 肖战被全网抵制那年，他29岁，刚红了半年 | 具体时间 + 让人代入的年龄 + 戏剧性反转 |
| 迪丽热巴 | 迪丽热巴红了十年，身边没有一个人护得住她 | "十年"时间感 + 情感缺口（想知道为什么） |
| 卢洋洋 | 卢洋洋在韩国当了9个月女团队长，然后一纸禁令全没了 | 具体时间+地点 + 戏剧性转折 |

### 标题规律总结

1. **数字是最强的信息碎片。** 上面7个标题，6个含具体数字。数字提供"硬信息感"
2. **反差/转折制造信息缺口。** "120公斤 vs 偶像剧男主""病床 vs 写专辑""刚红了半年 vs 全网抵制"
3. **人物全名必须在标题中出现。** 搜一搜 SEO 的硬性要求

### 坏标题改版案例

**孙正义篇的标题迭代：**

| 版本 | 标题 | 问题 |
|------|------|------|
| 初稿 | "孙正义这辈子，就没怕过输" | 情绪判断，没有信息缺口。像读后感标题 |
| 改版 | "听了6分钟吹牛就掏2000万，他赚回来1000倍" | 有了具体事实碎片 + 信息缺口 |

**教训：** 写完标题问自己——"这个标题里有没有一个具体事实？删掉人名，还有吸引力吗？"如果答案是否，重写。

## 结构案例

### 孙正义篇的结构改版

这篇经历了一次完整的从"传统人物稿"到"公众号爆款"的改版，教训最多。

| 维度 | 初稿（传统写法） | 改版（公众号写法） |
|------|-----------------|------------------|
| 开头 | 从1957年出生讲起 | 直接从"6分钟/2000万/580亿"最炸事实切入 |
| 结构 | 按时间顺序平铺（出生→求学→创业→成功→现在） | 先甩炸点 → 倒叙身世 → 回扣投资 → 暴跌 → 翻盘 |
| 代入感 | 全篇讲"他"，读者没有参与感 | 加入"放今天你敢不敢？""你我大概率做不到" |
| 结尾 | "把筹码推到桌子中间等开牌"（写完了的感觉） | 升维到"做大事的人和普通人的区别"，触发讨论/转发 |

**核心教训：写得好 ≠ 传得开。公众号要的是传得开。**

## 去 AI 味实战

### 最常犯的毛病（按出现频率排序）

1. **"不是A，是/而是B" 对仗句式** — 初稿中平均出现 3-4 次。全文只允许 1 次。写完 Ctrl+F 搜"不是""而是"
2. **每段都升华** — 板块结尾总是加"这意味着…""这说明了…"。改法：事实摆完，句号，下一段
3. **小标题太整齐** — 5 个标题全是同一句式。要混搭：问句、引号、感叹、长短交错
4. **AI 虚词** — "本质上""归根结底""从某种意义上说""值得注意的是"。人聊天不会用这些词，看到就删
5. **加粗太多** — 超过 8 处 = 没重点。砍到 5-6 处
6. **结尾排比收束** — "XX如此，YY如此，ZZ亦如此"。人收尾是随意的，不是对仗的
7. **模糊表达** — "部分创始人逐渐退出"是 AI 说法。"2018年黄江吉离开，新浪财经写了篇《小米最落魄创始人》"是人话

### Fix 实战

| 问题 | 怎么修 |
|------|--------|
| 板块间过渡生硬（话题突然跳转） | 在跳转前加一句主题桥接句。比如从巡演跳到婚姻，中间加"台上台下的XX，其实一直是同一个人" |
| 独立板块撑不起来（只有 2-3 段且信息模糊） | 不要硬立板块。核心信息融进相邻板块，用一两句话带过 |
| H2 缺少人名（SEO） | 选一个最自然的 H2 加上全名。"28岁出道"→"李荣浩28岁出道" |

## 搜索策略

### 第二轮补充搜索的 query 构造

第一轮用 pipeline 默认 query 拉广度数据。第二轮根据第一轮数据中最有故事性的线索构造聚焦 query。

**目标：补充直接引语、当事人原话、事件细节时间线、第三方媒体评价。** 这些是文章信息密度的来源。

**娱乐明星类必做第二轮**（信息量大、八卦多、细节多），至少 2 个聚焦 query。

Query 构造示例（根据实际数据变，不要照抄）：
- `"{人名} {某具体事件} 详细经过 当事人回应 原话"`
- `"{人名} 与 {某关键人物} 关系 合作 矛盾 细节"`
- `"{人名} {某年} 争议 媒体评价 网友反应"`

## 微信公众号兼容性踩坑

| 问题 | 原因 | 解法 |
|------|------|------|
| `<div>` 内联样式被剥离 | 微信编辑器对 div 特殊处理 | 信息框/强调框用 `<p>` 标签 |
| `background` 简写不生效 | 微信不支持 background 简写 | 必须用 `background-color` |
| 信息框字体变了 | 微信不从父容器继承 font-family | 信息框/强调框必须显式写 font-family |
| HTML 标签间空行变成空段落 | 微信编辑器把空行渲染成 `<p>` | 标签之间不要留空行 |
| `<section>` 多余间距 | 微信对 section 有默认样式 | 不用 section，用 p |
| `display:flex` 布局完全失效 | 微信编辑器不支持 flexbox | **不要用 flex。** 并排布局用 `display:inline-block`，简单列表用堆叠的 `<p>` |
| `display:flex` 改 `<table>` 后出现边框 | 微信编辑器给 table 加默认边框和间距 | **table 也别用。** 最可靠的方案是纯 `<p>` 堆叠 + `inline-block` |
| `linear-gradient` 不稳定 | 微信部分版本不渲染渐变 | 用纯色 `background-color` 替代 |
| `border-radius:50%` 的圆点不显示 | span 设了宽高但没 display:inline-block，或微信剥离了 | 不要用 CSS 画圆点，用 emoji（●）或文字符号代替 |
| `position:relative/absolute` 定位错乱 | 微信编辑器不支持定位布局 | 不要用 position 做布局 |
| `gap` 属性无效 | 微信不支持 CSS gap | 用 margin 代替 |

### 肖战篇的血泪教训（2026-04-15）

这篇文章在样式上反复踩坑，迭代了 4 个版本才放弃花哨样式：

1. **V1**：用 `display:flex` 做时间线、对比卡片、品牌矩阵、数据仪表盘 → 粘贴到微信编辑器全部错位
2. **V2**：改成 `<table>` 布局 → 微信给 table 加了默认边框，圆点 span 不渲染，更丑了
3. **V3**：改成纯 `<p>` + `inline-block` → 基本能用，但数据仪表盘三列还是不稳定
4. **V4（最终）**：放弃所有花哨样式组件，纯文字 + h2小标题 + 加粗。零兼容问题

**结论：微信公众号的 HTML 能力极其有限。不要做数据可视化、不要做多列布局、不要做卡片式UI。纯文字 + 信息框（`<p>` 带 background-color）+ 强调框 就是上限。把精力花在文字质量上，别花在样式上。**

## 数据获取策略

### 默认 query 和聚焦 query 的区别

**默认模板容易漏关键数据。** 肖战篇第一次用默认模板（"个人经历 重要事件 争议 最新动态"）拉数据，只拿到了人物经历和 227 事件，完全没有商业数据。第二次手动构造聚焦 query "肖战 商业版图 代言品牌 经纪公司关系 人脉"，才拿到了 15.8KB 的商业数据。

**教训：娱乐明星类必须做至少2轮搜索**（lessons.md 之前就有这条，但这次再次验证了）。第一轮拉广度，第二轮根据角度拉深度。如果要写"商业帝国"角度，query 里必须包含"商业""代言""品牌""经纪"等关键词。

### query 不要太复杂

聚焦 query 太长太具体反而搜不到东西（API 返回 0.1KB 空结果）。**query 控制在 15-20 个字以内**，用空格分隔 4-5 个关键词就够了。

## 写作角度选择

### "经纪关系"是娱乐明星类的好切入点

肖战篇的核心角度是"经纪公司管不了他"——这个角度好在：
1. 反常识：普通人以为签了经纪公司就被公司管着
2. 能串联商业数据：为什么高奢、为什么不铺量、为什么敢沉默
3. 有叙事张力：从设计师到独立运营的成长线

**但不是每个艺人都适合这个角度。** 只有经纪关系确实松散/特殊的艺人才适合。

### 标题含人名很重要

用户明确反馈：标题必须有主角名字。"从甲方爸爸到Gucci全球代言人"被打回，改成"肖战：从甲方爸爸到Gucci全球代言人"才通过。**这也是 SEO 的硬性要求，不能省。**

## 肖战十日终焉篇的完整教训（2026-04-16）

这篇文章从选角度到定稿经历了两天、两个完全不同的版本，踩的坑比之前所有文章加起来都多。

### 标题：AI 写不出好标题，别硬磨

肖战篇的标题迭代了 **8 轮以上**，用 AskUserQuestion 给了 20+ 个选项，用户全部打回，最终用户自己写了标题。

| 阶段 | 示例 | 用户反馈 |
|------|------|---------|
| AI 初稿 | "肖战从咖啡馆logo画到了十日终焉" | "从A到B"是 AI 句式，一眼假 |
| AI 变体 | "肖战画了十年，最新一笔叫十日终焉" | 概念好但句式不对味 |
| AI 再磨 | 各种排列组合 | 用户连续4次选"都不行"，说"说不清，就是感觉不对" |
| 用户自己写 | "肖战第一笔是咖啡馆，最新一笔叫十日终焉" | 通过 |

**教训：**
1. AI 生成标题超过 3 轮还没过，**别继续磨了**。直接问用户"你心里大概想要什么感觉？给我几个关键词或者半成品，我来调"
2. "从A到B""A是X，B是Y"这类对称结构是 AI 标题的重灾区，尽量避免
3. 用户说"说不清"的时候不要追问"哪里不对"，直接给更多选项或者换思路
4. 好标题的"味道"是玄学，AI 能提供素材（关键词、概念），但最终组装往往需要人来

### 去 AI 味：新发现的 AI 腔模式

以下是本次对话中用户新指出的 AI 模式，**追加到 gen_article.md 的 8 大毛病之外**：

| # | AI 腔模式 | 用户原话 | 为什么是 AI 味 |
|---|----------|---------|--------------|
| 9 | "但很少有人认真想过" | "非常的古法" | AI 喜欢用"大家都忽略了"来引出自己的分析，人不会这么说 |
| 10 | "做过XXX的都懂" | "这种就是 AI 体" | 假装圈内人的万能句式，用多了就假 |
| 11 | "同一天，XXX也..." | "同一天，这种" | AI 爱用时间巧合制造戏剧感，但真人写稿很少这么用 |
| 12 | "砍掉的是XXX" | "砍掉的xx都是ai体" | 名词化+排比列举，人不会把"去掉前缀"说成"砍掉的是团体前缀、英文昵称、过去十年的所有标签" |
| 13 | "没了...没了...没了" | "还是ai味" | 并列"没了"是 AI 排比的变体 |
| 14 | "正式落幕" | 连带上面一起被指出 | 新闻通稿用语，粉丝文章不会用这么正式的词 |
| 15 | "这意味着什么"反复出现 | 金莎五金篇指出 | AI 爱用同一句式反复解释，人不会老问"这意味着" |
| 16 | 口号体堆叠 | "姐就是值得""姐妹们清醒一点""这才是最顶级的" | 单独看没问题，连着用就太整齐，像AI排比 |
| 17 | 信息框里四句排比 | "不要因为...不要因为...不要因为...不要因为..." | 太工整，人话应该是散的，三句+转折，或者长短错开 |
| 18 | 小标题全是同一句式 | 五个标题都是"XX是XX" | 要混搭：问句、引号、感叹、纯陈述 |

**核心规律：AI 腔的本质是"过度整齐"。** 不管是对仗、排比、并列还是时间呼应，只要读起来太工整太有设计感，就是 AI 味。人写东西是散的、随意的、不追求对称的。

### 角度选择：同一份数据可以写出完全不同的文章

肖战这个 topic 写了两个版本，角度完全不同：

| 版本 | 角度 | 结果 |
|------|------|------|
| V1（04-15） | 命运互文：肖战和齐夏的人生镜像 | 被否。太文学化，像影评不像粉丝文 |
| V2（04-16） | 设计师底色：用"改稿"串联肖战所有选择 | 通过。"你好懂肖战"的感觉出来了 |

**教训：**
1. 粉丝文章的角度要让粉丝觉得"你懂他"，不是让路人觉得"分析得好"
2. "命运互文""人生镜像"这类文学手法太高冷，粉丝文不适合
3. 好角度的标准：能用一个具体的、粉丝知道但没深想过的事实当锚点（比如"肖战是设计专业毕业的"），然后用这个锚点重新解读他的所有选择

### 结尾：不要求关注，不要文艺升华

结尾回到人物本身——他的性格、选择、执念。结尾就是结尾，不加"点个关注"之类的行动指令，也不做人生道理式的升华。读者读完有感觉，自然会关注；强行求关注反而打断阅读体验。

### 字数：娱乐明星类宁长勿短

初稿约 1800 字，用户要求扩到 3000 字。娱乐明星类数据丰富、引语多、故事细节多，**写短了反而像没诚意**。3000 字是底线，如果素材够好可以到 3500。

扩写不是注水，是加更多的：
- 当事人原话和采访引语
- 具体时间地点人物的事件细节
- 粉丝熟悉但路人不知道的冷知识

**操作规范：初稿写完必须用脚本验字数。** 统计方式：去掉 HTML 标签和 script 后统计中文字符+标点。低于 2500 字必须扩写，不发。

```python
import re

def count_chinese(html_file):
    with open(html_file, 'r', encoding='utf-8') as f:
        content = f.read()
    content = re.sub(r'<script.*?</script>', '', content, flags=re.DOTALL)
    content = re.sub(r'<[^>]+>', '', content)
    return len(re.findall(r'[\u4e00-\u9fff，。！？、：；""''「」【】《》…—]', content))
```

### 选题情绪方向：获得型 vs 消耗型（2026-04-20 验证）

**悲剧选题写得再好，读者也不想打开。**

大S去世的故事写得工整、信息密度达标，但用户看完第一句就知道情绪基调是"沉重"，不想付出那个情感成本。

选题时增加一个自检维度：

| 维度 | 获得型（推荐） | 消耗型（慎选） |
|------|-------------|-------------|
| 情绪方向 | 有趣/爽/好奇/共情 | 悲剧/遗憾/沉重 |
| 读者行为 | 主动想知道后续 | 被动接受坏消息 |
| 打开驱动 | "这个我不知道" | "要不要消耗情感" |
| 典型例子 | 何润东项羽翻红、张凌赫120公斤 | 大S去世、艺人离世 |

**原则：零粉丝阶段优先选获得型。** 消耗型题材需要读者对这个人已经有感情基础，才愿意付出情感成本。没有粉丝基础的账号写悲剧，等于让陌生人替你难过——大概率不干。

## 选题策略：人物热度 vs 信息缺口

### 顶流是红海，上升期人物是蓝海（2026-04-16 数据验证）

李昀锐（上升期）的文章阅读量 > 肖战（顶流）的文章。原因不是写得好坏，是选题的结构性差异：

| 维度 | 上升期人物（李昀锐） | 顶流（肖战） |
|------|---------------------|-------------|
| 读者已知信息 | 几乎为零，每个信息点都是新的 | 粉丝倒背如流，路人也大概知道 |
| 搜一搜竞争 | 蓝海，文章容易排前面 | 红海，几万篇文章抢同一个关键词 |
| 标题信息缺口 | 大——"跑丢4个摄像师""985""运动员当演员"都让人想点 | 小——"设计师→央视冠军"，大家已经知道他成功了 |
| 文章价值 | **告诉你一个你不了解的人** | 帮你重新理解一个你已经了解的人 |

**核心规律："告诉你不知道的事"永远比"帮你重新理解已知的事"传播力强。**

### 好故事 > 大流量（卢洋洋篇验证）

卢洋洋（冷门）阅读量 > 肖战（顶流）。甚至比李昀锐（上升期）还好。原因：

1. **标题自身就是一个完整的戏剧弧线**："卢洋洋在韩国当了9个月女团队长，然后一纸禁令全没了"——有了→没了，三重信息缺口（韩国女团？9个月？什么禁令？），知道她的人和不知道的人都想点
2. **故事有"命运感"**：师范生→被拉去比赛→懵懂被选中→去韩国→刚火就被萨德一刀切→回国从零→六年磨成司清。"什么都不是她能控制的，但她没停"——这种故事天然让人想转发
3. **搜一搜零竞争**：可能是唯一一篇完整串联她从女团到演员的文章
4. **每个板块结尾都有钩子**：不像肖战篇中后段变成成绩罗列，卢洋洋篇每段结尾都让人想读下一段

**核心规律：一个冷门人物的好故事，传播力远超一个顶流的成绩单。选题时先问"这个人的故事有没有命运感"，再问"这个人有多少粉丝"。**

### 对零粉丝账号的选题建议

1. **优先选"正在上升但还没被写烂的人"**——有热度但搜一搜竞争低，信息缺口天然大
2. **冷门人物如果故事够好，比上升期人物还能跑**——关键是故事本身有戏剧弧线和命运感
3. **顶流可以写，但要有心理预期**——你是在高手如林的红海里搏存在感，阅读量大概率不如前两类
4. **如果非要写顶流，角度必须足够刁钻**——不能是粉丝已经知道的故事的重新排列，必须有一个"连老粉都不知道"的切入点

### 核心认知：选题决定80%，写作只决定20%（2026-04-17 复盘）

肖战篇投入了最多的精力——角度迭代2版、标题磨了8轮、去AI味反复打磨、多轮补充搜索。李昀锐篇几乎没怎么看就发了。结果：李昀锐阅读量 > 肖战。

**这不是写作质量的问题，是优化了错误的变量。**

零粉丝账号的阅读量公式：

```
阅读量 ≈ 标题点击率 × 搜一搜竞争位 × 信息缺口大小
```

- **标题点击率**：取决于标题里有几个信息钩子（数字、反常识、悬念），跟文章本身写得好不好无关
- **搜一搜竞争位**：取决于这个人名有多少竞品文章，跟你的文章质量几乎无关
- **信息缺口大小**：取决于读者对这个人已经知道多少，这是选题决定的

文章质量影响的是**转发率**和**完读率**——但零粉丝账号没有初始流量池来放大转发，所以文章质量的权重很低。

**结论：把"深思熟虑"花在选题和标题上，而不是花在文章打磨上。** 一篇选题对的文章随手写完发出去，大概率跑赢一篇选题错的文章反复打磨三天。

**操作指南：**
1. 选题花60%的时间：这个人的搜一搜竞争有多激烈？读者的信息缺口有多大？
2. 标题花20%的时间：有没有具体数字？不认识这个人的人会不会想点？
3. 写作花20%的时间：信息密度够、去AI味过一遍、就发

#### 规律一：同一个人，标题差6倍

肖战三篇（同一个号、同一周）：

#### 规律二：阅读量和分享率是两个独立指标

- 王子杰：137阅读 / 11分享（分享率**8%**，全场最高）→ 文章好但搜一搜没入口

#### 规律三："从A到B"式标题是毒药（数据验证）

"从甲方爸爸到Gucci全球代言人" 264，"第一笔是咖啡馆，最新一笔叫十日终焉" 168。变化型叙事缺冲突缺悬念。看到"从A到B"就改掉。

#### 规律四：爆款标题 = 具体数字 + 反转/冲突

Top 5 无一例外：9个月+全没了、4个摄像师+985运动员去当演员、29岁+刚红半年就被抵制、3000万张+品牌全跑了、十年+没人护得住。有数字没反转（"15亿的秘密"）只能到147。

#### 规律五：低阅读量的共同特征

85-165阅读的文章（李荣浩、叶舒华、阚清子、孟子义、孙怡）：
- 人物搜索热度低（搜一搜没入口）
- 或标题虽好但人物在国内知名度不够（叶舒华）
- 或标题数字对仗太整齐有AI味（"出道十年被黑八年，第九年她红了"）

**搜一搜热度是底线。人物没热度，标题再好也没用。** 写之前先在微信搜一搜里搜一下人名，看有没有搜索联想。

每篇文章不要套同一个模板，根据人物调性换风格：

## 流量型文章写作经验（2026-04-19，5篇热搜文章验证）

### 标题：具体数字 + 反转/冲突 + 信息缺口

**爆款标题公式（数据验证）：**

| 话题 | 标题 | 元素分析 |
|------|------|----------|
| 谢娜张杰小火车 | 谢娜登张杰鸟巢小火车：每场只抽8人，她等了一周才上车 | 具体数字8人+一周+反转"让掉了等了一周" |
| 虞书欣父亲索赔 | 虞书欣父亲索赔17万败诉：告高三学生，法院说是公众监督 | 具体数字17w+身份对比"成年人vs高三学生"+反转"败诉" |
| 沈梦辰热玛吉 | 沈梦辰连续4年做热玛吉：脸越打越薄，她说这是骨相美 | 具体数字4年+反常识"越打越薄"+反转"说是美" |
| 肖战压力 | 肖战说压力没那么大：125天拍101天，他凭什么不焦虑 | 具体数字125/101+冲突"顶流说不焦虑" |
| 杜华雷军 | 杜华追星雷军：说很帅很nice，网友说这组合太有梗了 | 直接引语+跨界碰撞"娱乐圈vs科技圈" |

**核心规律：**
1. 数字必须具体（8人、17万、4年、125天、101天），不能用"很多""少量"这种模糊词
2. 必须有反转或冲突（让掉了、败诉、越打越薄、顶流说不焦虑）
3. 信息缺口要大（让读者想知道"然后呢""为什么"）

### 去 AI 味：转发引导框的写法

**错误写法（太整齐）：**
```html
姐妹们，转发给那个不敢要金的人。

有人需要听到这句话：你值得那套别人眼里是虚荣、你眼里是底气的金。
```
问题：两句话都是排比结构，太AI。

**正确写法（散一些）：**
```html
转发给那个正在降低标准的她。

有人需要听到这句话：你值得那套别人眼里是虚荣、你眼里是底气的金。
```
改进：去掉"姐妹们"，把"不敢要金"改成"降低标准"，更自然。

### 伪女权/男女对立话题的写作要点

**核心逻辑：**
- 表面：吹捧女人、强调女人值得最好的
- 实际：制造焦虑、刺激讨论、获取流量

**写作技巧：**
1. 用具体事件（五金、热玛吉）作为切入点
2. 制造对比（真夫妻vs假CP、成年人vs未成年人）
3. 用"值得""底气""诚意"等词包装物质诉求
4. 结尾用转发引导，但不要太整齐

**避坑：**
- 不要用"姐妹们"开头（太AI）
- 不要用"要金""要五金"这种索取感词汇（改用"降低标准""不敢坚持"）
- 信息框里的排比不要超过3句，长短要错开

### 标题中的人名处理

**娱乐明星类文章：**
- 如果是男方视角（如张杰），标题可以不放女方全名
- 如果是女方视角（如沈梦辰），必须放全名（SEO）
- 事件中心人物必须放全名（虞书欣父亲、肖战）

**例外：**
- 夫妻档可以只放一个人名（"谢娜登张杰小火车"而非"谢娜张杰小火车"）
- 跨界组合优先放娱乐圈一方（"杜华追星雷军"而非"雷军杜华"）

### 搜索数据保存规范

每次搜索后必须保存到 `data/searchdata/{日期}_{话题}_{source}.md`，包含：
- 搜索时间、来源、关键词
- 事件概述（5-10句话）
- 核心信息点（分条列出）
- 标题方向（3个备选）
- 文章角度（1个核心角度）
- 情绪触发点（5个左右）

**每篇文章颜色必须不同**，避免读者连续看5篇觉得"怎么都一样"。

## 微信限流诊断与突破（2026-04-20）

### 限流的本质：两套独立问题

微信对小账号有两种限流，机制完全不同，解法也不同，不能混淆：

| 类型 | 表现 | 根本原因 | 解法方向 |
|------|------|---------|---------|
| **内容降权** | 文章发出去，搜一搜搜不到 / 推荐极少 | AI味触发检测降权，或内容质量分低 | 去AI味、提升完读率和分享率 |
| **账号限流** | 卡在1000阅读上下，换什么内容都一样 | 账号级别限制，新号冷启动期 / 历史行为触发 | 时间+持续发布+互动积累 |

**先判断是哪种。** 如果每篇文章阅读量都稳定卡在同一个数值附近，是账号级别问题；如果不同文章差异很大，是内容级别问题。

---

### 内容降权：AI检测是核心威胁

微信已集成腾讯朱雀AI检测算法。**AI浓度超过60%会被自动降权，阅读量可能暴跌50%-90%。安全线是20%以下。**

我们的去AI味流程已经在做，但需要升级到"发布前必过检测"：

**发布前必做（新增强制步骤）：**
1. 用腾讯朱雀检测扫一遍（免费20次/天，地址：`https://zhuque.tencent.com`）
2. AI浓度 > 20%：重做去AI味改写，重点是打散句式、加口语、加具体细节
3. AI浓度 ≤ 20%：发

**去AI味检查和AI检测是两件事。**

**敏感词触发 Review 不是 AI 问题：** 廖子妤文章因描述"全裸演出"被 TMS 标为 `Review / Porn`。解法是把涉及尺度描述改成间接表述（"对角色有突破性的演绎"）。凡是文章涉及演员大尺度表演、亲密戏份、争议事件，用词要模糊化，让读者自行理解。 我们的去AI味规范是为了读者体验，AI检测是为了过平台算法。两者高度相关但不等同——有些平台检测不出来的内容读者也会感觉AI味重，有些检测率高的内容读者不觉得有问题。都要过。

---

### 搜一搜排名：权重分布和可操作点

2025年搜一搜算法五个维度权重：

| 维度 | 权重 | 我们能控制什么 |
|------|------|-------------|
| 内容相关性 | 35% | 标题/前120字/h2含人名，关键词自然密度 |
| 账号权威性 | 25% | 持续更新（断更2周掉30-50%权重）、原创标签 |
| 用户行为 | 20% | 转发 > 收藏 > 完读率，这是文章质量决定的 |
| 社交影响力 | 15% | 群聊传播、好友关注，短期难干预 |
| 时效性 | 5% | 发布时间，热搜文要快 |

**可立即执行的动作：**
- 保持每周2-3篇固定更新节奏，不断更
- 每篇文章发出后，自己在评论区留一条有实质内容的评论（不是"写得好"，是补充一个文章没写的细节），引导读者互动
- 主动把文章分享到2-3个相关微信群（真实讨论型的群，不是广告群），第一波阅读量来源影响搜一搜权重

---

### 账号级限流（卡1000阅读）：时间换空间

如果是账号级限流，没有捷径。2025年9月和2026年1月的两轮大限流，小账号普遍被卡在1000阅读，这是平台对新号/低权重号的冷启动限制。

**解法：**
- 持续发布，不停更（停更会进一步降权）
- 每篇文章的完读率和分享率是账号权重累积的来源
- 积累到一定历史数据后，平台会重新评估账号权重
- 预期周期：通常3-6个月才会有明显改善

**不要做的事：**
- 刷量（1500元一万阅读，平台能识别，封号代价远大于收益）
- 短期内频繁删文重发（删文被判定为低质量信号）
- 每天超过3篇发布频率（被判定垃圾营销触发限流）

---

### 选题和标题对限流的影响

即使在限流期，不同文章的阅读量差异可以达到3-5倍。这部分差异来自搜一搜——限流影响的是推荐分发，不影响搜索分发。

**限流期的核心策略：押注搜一搜，放弃推荐流。**

搜一搜能给流量的前提：
1. 人物在微信搜一搜有搜索热度（先搜一搜确认有搜索联想词）
2. 文章标题和前120字命中搜索关键词
3. 文章发出后有真实的阅读和转发行为（影响搜一搜排名）

**每篇文章发出后的标准动作（新增）：**
- 发到自己的朋友圈（一次真实曝光）
- 转发到1-2个相关群（冷启动流量）
- 自己在评论区留1条有实质内容的评论
- 等48小时看搜一搜是否能搜到文章

---

### 原创标签：必须开

原创标签可增加搜索优先级约30%，且是账号权威性的重要信号。每篇文章发布时必须勾选原创。

---

### 总结：限流期行动清单

**发布前：**
- [ ] 腾讯朱雀检测，AI浓度 ≤ 20%
- [ ] 搜一搜确认人名有搜索热度
- [ ] 标题和前120字含人名+关键词
- [ ] 勾选原创标签

**发布后：**
- [ ] 转发朋友圈
- [ ] 转发1-2个真实相关群
- [ ] 评论区留1条有实质内容的评论
- [ ] 48小时后检查搜一搜能否搜到

**账号层面：**
- [ ] 每周2-3篇，固定节奏，不断更
- [ ] 不刷量，不频繁删文，不超过每天3篇

## 选题：从热搜边缘找配角（2026-04-20 验证）

### "没有新意"的诊断

第一批选题：迪丽热巴（顶流，故事被写烂）、鹿晗关晓彤（分手传闻持续半年，竞品极多）、章子怡（今日落败金像，但她的故事信息缺口为零）。用户反馈"没有新意"。

**根本问题：选了热搜的主角，而主角永远是竞争最激烈的位置。**

正确的找法：**去热搜的边缘，找今天被提到但没人认真写的配角。**

| 今日热搜主角 | 被忽视的配角 | 配角的故事潜力 |
|------------|------------|-------------|

### 初稿天然偏短：为什么以及怎么补

这次三篇初稿字数：孙丞潇1395字、廖子妤1243字、大S1464字。全部不足3000字底线。

**初稿偏短的系统性原因：**
AI写初稿时天然倾向于"把核心故事说完就停"，不会主动补充背景、对比、人物周边关系。这些补充对读者理解很重要，但不是"核心故事"的一部分，所以AI不会自发写。

**扩写的四种合法手段（不是注水）：**

| 手段 | 具体做法 | 例子 |
|------|---------|------|
| **补人物背景** | 主角出现前，先交代"另一方"的处境 | 孙丞潇篇：先写金莎的处境（42岁，淡出，无人关注），再写孙丞潇出现 |
| **补时间轴上的空白** | 两个关键事件之间发生了什么 | 拿女配后到病了两年这段，廖子妤为什么没有立刻进入高峰 |
| **加对比维度** | 这件事对同类人意味着什么 | 上戏毕业的人正常路径是什么，孙丞潇的路和正常路径差了多少 |
| **加一个只有深度搜索才能找到的细节** | 陈家乐NG20次、导演找了五六年演员、金莎攒五金时金价才四五百 | 这些细节来自第二轮搜索，初稿时没有 |

**不合法的注水（让文章变差）：**
- 把已经说过的事换个说法再说一遍
- 加分析和感想（"这说明了…""我们可以看到…"）
- 凑数的过渡段落

### 通读Review能发现脚本发现不了的问题

脚本只检查：字数、加粗次数、h2人名、AI虚词、对仗句式出现次数。

**脚本发现不了的问题，必须靠人工通读：**

1. **同一句话在不同板块重复出现**（廖子妤篇："全副身家不到5000港元"出现两次；导演评价"天才"出现两次）
2. **同一个事实用不同措辞说了两遍**（冰岛求婚的金价细节在两个板块各说了一次）
3. **板块之间逻辑不顺**（从"他后来真的演了男主角"跳到"冰岛求婚"，读者感觉突兀）
4. **结尾过长，重复盘点**（廖子妤最后三段，把同一个主题"她很踏实，一步一步走来"说了三种方式）

**操作规范：脚本通过后，必须再做一次人工通读，专门找重复。**
快速找重复的方法：搜索文章里关键名词第一次出现的位置，看同一个词/概念在正文里出现了几次，超过2次的检查是否重复。
