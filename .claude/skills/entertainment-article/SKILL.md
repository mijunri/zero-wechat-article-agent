# Entertainment Article - 娱乐公众号写作

专门用于撰写娱乐类公众号文章的技能，基于16篇文章数据复盘总结。

## When to use

- 用户想写娱乐类公众号文章
- 用户提到"热搜""明星""八卦""流量文"
- 用户需要根据热点事件创作内容
- 用户想写明星/艺人相关文章
- 用户需要爆款标题和流量导向的内容

---

## 必须执行的生产流水线（不允许跳步）

```
第1步：选人（不是选热搜）
    ↓
第2步：多轮搜索，建立数据文件
    ↓
第3步：选角度（读完全部数据再决定写什么）
    ↓
第4步：按写作规范写初稿
    ↓
第5步：去AI味改写
    ↓
第6步：Review清单逐项检查
    ↓
第7步：输出HTML
```

**核心问题诊断：文章数据差的根本原因不是"写得不好"，是流程断了——跳过搜索和数据整理，直接从热搜标题起笔写评论。热搜评论文是"有观点没信息"，目标是人物故事文："有信息没说教"，让事实说话。**

---

## 第1步：选人逻辑

**不是"今天谁上热搜写谁"，而是"这个人背后有没有一条完整的、读者不知道的故事线"。**

### 选题优先级

1. **上升期但还没被写烂的人** — 有热度但搜一搜竞争低，信息缺口天然大
2. **冷门人物但故事够好** — 关键是故事本身有戏剧弧线和命运感（已验证：冷门人物的好故事 > 顶流的成绩单）
3. **顶流** — 最后选择，必须有刁钻角度，不能是粉丝已经知道的故事的重新排列

### 选题自检（写之前先问自己）

- 这个人的搜一搜竞争有多激烈？（微信搜一搜里搜人名，看有没有搜索联想、竞品文章多不多）
- 读者的信息缺口有多大？（读者已经知道多少？你能告诉他们什么新东西？）
- 这个人的故事有没有"命运感"？（有起有落有转折，不是平铺直叙的成功故事）

### 进阶：从热搜边缘找配角

**热搜主角永远是竞争最激烈的位置。** 更好的策略是去找今天被提到、但没人认真写的"配角"。

操作方法：看今天的热搜事件，找事件里的次要人物：

| 热搜事件 | 主角（竞争激烈） | 配角（蓝海） |
|---------|--------------|-----------|
| 金莎宣布婚讯 | 金莎 | 孙丞潇（小19岁新郎，故事没人写） |
| 金像奖颁奖 | 章子怡（落败） | 廖子妤（获奖，内地读者几乎不认识她） |
| 逐玉热播 | 张凌赫 | 何润东（没出演却靠14年前角色翻红） |

判断配角值不值得写：
1. 这个配角身上有没有读者完全不了解的东西？
2. 这个人的故事有没有完整弧线（起→转折→结果）？
3. 搜一搜里这个人的竞品文章多不多？

### 零粉丝账号选题建议（16篇数据验证）

| 选题类型 | 优势 | 注意 |
|---------|------|------|
| 上升期人物 | 搜一搜蓝海，信息缺口大，每个信息点都是新的 | 先确认搜一搜有搜索联想 |
| 冷门人物+好故事 | 零竞争，命运感强的故事天然传播 | 故事必须有完整弧线：起→转折→低谷→重生 |
| 顶流 | 自带流量 | 红海，标题必须够刁，文章价值必须是"告诉你不知道的事"而非"帮你重新理解已知的事" |

**核心规律：选题决定80%，写作只决定20%。**

---

## 第2步：多轮搜索，建立数据文件

**娱乐明星类必须做至少2轮搜索。** 第一轮拉广度，第二轮根据第一轮数据中最有故事性的线索构造聚焦 query。

### 第一轮（广度）

query 示例：
- `"{人名} 个人经历 出道 背景"`
- `"{人名} 重要事件 争议 转折"`

### 第二轮（深度）

根据第一轮结果构造，目标是补充直接引语、当事人原话、事件细节时间线、第三方媒体评价：

- `"{人名} {某具体事件} 详细经过 当事人回应 原话"`
- `"{人名} 与 {某关键人物} 关系 合作 矛盾 细节"`
- `"{人名} {某年} 媒体评价 采访"`

**query 控制在 15-20 字以内，用空格分隔 4-5 个关键词。太长太具体反而搜不到。**

### 数据文件保存规范

搜索完毕后必须保存到 `data/searchdata/{日期}_{话题}_{source}.md`，必须包含：

- 来源（API/URL）
- 获取时间
- 查询关键词
- 原始内容
- 引用链接
- 核心信息点（分条列出）
- 标题方向（3个备选）
- 文章角度（1个核心角度）
- 情绪触发点（5个左右）

## 第3步：选角度

**先读完全部数据，再决定这篇文章写什么。不要默认写人脉分析或热搜评论。**

问自己：这份数据里，哪个信息让我自己都"卧槽"了一下？那个就是切入点。

### 可能的角度

- 一个反常识的事实（"所有人以为他是XX，其实他是YY"）
- 一个关键的转折决策（"他在XX年做了一个所有人反对的决定"）
- 一段不为人知的经历或关系
- 一个有意思的矛盾（"他最大的优势恰恰是他最大的风险"）
- 一个让人唏嘘的变迁（"当年的XX人，如今只剩XX"）

选定角度后，整篇文章围绕这一个角度展开。不要贪心，不要试图面面俱到。

## 第4步：写初稿

### 文章类型

| 类型 | 字数 | 适用场景 |
|------|------|----------|
| 长篇深度 | 2500-3500字 | 人物全景、事件复盘、深度扒料 |
| 短篇快讯 | 1000-1500字 | 单一事件、热点速递、一个反转 |

**默认写长篇。** 用户指定"短篇""快讯""短文"时写短篇。娱乐类素材丰富宁长勿短。

### 写作人格

你是一个**懂行的欣赏者/圈内人**。你真正了解这个人，用带点幽默的方式跟读者讲他的故事。你有自己的判断，对争议不回避但有立场——立场是理解而非批判。保持记者的信息素养：有出处、有细节、有时间线。

- 目标读者：明星粉丝 + 路人。粉丝读完想转发，路人读完想入坑
- 情感立场：以欣赏和理解为底色，不是中立旁观
- 阅读时长：5-7分钟（约2500-3500字）

### 标题铁律

**爆款标题公式 = 具体数字 + 反转/冲突**

| 话题 | 标题 | 元素分析 |
|------|------|----------|
| 卢洋洋女团 | 卢洋洋在韩国当了9个月女团队长，然后一纸禁令全没了 | 具体时间+地点+戏剧性转折 |
| 谢娜张杰小火车 | 谢娜登张杰鸟巢小火车：每场只抽8人，她等了一周才上车 | 8人+一周+反转 |
| 虞书欣父亲索赔 | 虞书欣父亲索赔17万败诉：告高三学生，法院说是公众监督 | 17w+身份对比+反转"败诉" |
| 沈梦辰热玛吉 | 沈梦辰连续4年做热玛吉：脸越打越薄，她说这是骨相美 | 4年+反常识+反转 |
| 肖战压力 | 肖战被全网抵制那年，他29岁，刚红了半年 | 具体时间+年龄+戏剧性反转 |
| 虞书欣专辑 | 虞书欣专辑卖了3000万张，四个月后品牌全跑了 | 数字+反转 |

标题铁律：
1. **必须有"硬信息"** — 数字、人名、具体事件
2. **信息缺口要大** — 读者知道大概在聊什么，但不知道具体是什么
3. **15字以内最佳，最多22字**
4. **必须含人物全名**
5. **每篇标题必须独特**

禁止的标题类型：
- "从A到B"式（数据验证：是毒药，流量差6倍）
- 情绪判断/概括总结（"XX这辈子就没怕过输"）
- 模板体（"扒完XX的XX，我沉默了"）

### 文章结构

**没有固定的段数，没有固定的格式。** 根据角度和素材自然组织。唯一的要求：读者读完每一段都想读下一段。

**开头：3秒定生死**
- 第一句话直接甩最炸的事实（时间+人物+具体数字+反常识行为）
- 前三段之内必须抛出悬念或反转
- 禁止："说到XX大家可能都知道"、从出生讲起、任何概括性人物介绍

**中间：过山车叙事**
- 先甩最炸的点 → 倒叙/插叙解释为什么 → 再甩第二个炸点 → 收束
- 小标题要有态度，句式混搭（问句、引号、感叹、长短交错，至少2种）
- 板块节奏：有长有短（至少1个板块只有2-3句话）
- 不是每个板块都需要"分析"，有时候事实摆出来读者自己品

**信息密度（每个核心板块）：**
- ≥1 个直接引语或对话（当事人原话、采访原文）
- ≥1 个具体数字（金额、时间、人数、排名）
- ≥1 个来源佐证（媒体名、当事人名）
- 优先用原始细节，不要概括。数据里有"没打麻药，插了7根引流管，缝了80多针"，就别写成"受伤很严重"

**代入感：** 至少1-2处直接和读者对话，用"你"字。幽默像老粉之间的会心一笑，不是旁观者的讽刺。

**结尾：**
- 回到人物本身——他的性格、选择、执念
- 不要升华到"人生道理"
- 结尾回到人物本身，不要求关注，不要文艺升华
- 不要"综上所述"，不要排比收束

### 初稿字数不足时的扩写手段

初稿天然偏短（AI倾向于"把核心故事说完就停"）。扩写到3000字要靠这四类内容，不是注水：

| 手段 | 做法 | 例子 |
|------|------|------|
| **补"另一方"的背景** | 主角出现前，先交代对手/配偶/搭档的处境 | 孙丞潇篇：先写金莎淡出多年的处境，再写孙丞潇出现 |
| **补时间轴空白** | 两个关键事件之间发生了什么 | 廖子妤拿女配→病了两年这段空档 |
| **加对比维度** | 这件事对同类人意味着什么 | 上戏毕业的人正常路径是什么，孙丞潇的路差了多少 |
| **加第二轮搜索独家细节** | 只有深度搜索才找到的信息 | 陈家乐NG20次、导演找演员找了五六年、金莎攒五金时金价才四五百 |

**不合法的注水（让文章变差）：**
- 把已说过的事换个说法再说一遍
- 加分析感想（"这说明了…""我们可以看到…"）
- 凑数的过渡段

## 第5步：去AI味（最关键的一步）

**写完初稿后，必须做一轮完整的去AI味改写。** AI腔的本质是"过度整齐"——只要读起来太工整太有设计感，就是AI味。人写东西是散的、随意的、不追求对称的。

### 14大AI腔模式（必须检查）

| # | 毛病 | 表现 | 改法 |
|---|------|------|------|
| 1 | 对仗句式上瘾 | "不是A，是/而是B"反复出现 | 全文不超过1次 |
| 2 | 结尾排比收束 | 三个短句并列+最后转折 | 人收尾是随意的，两句散的比三句齐的更真 |
| 3 | 每段都升华 | 板块结尾"这意味着…""这说明了…" | 事实摆完，句号，下一段 |
| 4 | 小标题太整齐 | 5个标题全是同一句式 | 混搭至少2种 |
| 5 | 结构均匀 | 板块长度一样写法一样 | 至少1个板块只有2-3句话 |
| 6 | AI虚词 | "本质上""归根结底""值得注意的是" | 看到就删 |
| 7 | 加粗太多 | 超过8处 | 砍到5-6处 |
| 8 | 模糊表达 | "部分创始人逐渐退出" | 换成具体事实+人名+时间 |
| 9 | "但很少有人认真想过" | AI用"大家都忽略了"引出分析 | 人不会这么说，删 |
| 10 | "做过XXX的都懂" | 假装圈内人的万能句式 | 用多了就假，删 |
| 11 | "同一天，XXX也..." | AI用时间巧合制造戏剧感 | 真人很少这么写，删 |
| 12 | "砍掉的是XXX" | 名词化+排比列举 | 改成自然叙述 |
| 13 | "没了...没了...没了" | 并列排比变体 | 改散 |
| 14 | "正式落幕" | 新闻通稿用语 | 换口语 |

### 通用去AI味原则

- 打破完美结构：不要"首先/其次/最后"
- 加不完美元素：口语化（"说实话""老实讲"）
- 长短句混搭
- 连接词替换："此外"→"还有啊"；"然而"→"但问题是"

### 去AI味自检操作

1. 搜"不是"和"而是" → 对仗句式不超标
2. 检查结尾 → 没有排比收束
3. 每个板块结尾 → 删多余判断句
4. 小标题句式 → 至少2种不同风格
5. 板块长度 → 有短板块（2-3句话）
6. 搜"本质上""归根结底" → 全删
7. 数加粗 → 不超过6个
8. 搜"部分XX""逐渐XX" → 换成具体事实

## 第6步：Review清单（写完必过一遍）

### 质量
- [ ] 每个板块能独立成立？信息模糊的板块融进相邻板块
- [ ] 每个核心板块：≥1直接引语 + ≥1具体数字 + ≥1来源佐证
- [ ] 数据利用率 ≥ 60%（对照数据文件）
- [ ] "不是A，是/而是B"全文不超过1次
- [ ] **人工通读找重复**：同一句话/同一个评价/同一个细节是否在不同板块出现了两次（脚本检测不出来）

### 人格一致性
- [ ] 全篇是"懂行的欣赏者"，不是"中立旁观者"或"批评者"
- [ ] 对争议是帮读者理解，不是评判对错
- [ ] 幽默来自喜爱，不是讽刺

### 流量导向
- [ ] 标题含具体事实碎片（数字/反转/冲突）
- [ ] 第一句话就是最炸的事实
- [ ] 前三段有悬念或反转
- [ ] 不是时间顺序平铺，有倒叙/插叙
- [ ] 至少1-2处用"你"字和读者对话

### 去AI味
- [ ] 小标题至少2种风格
- [ ] 有1个短板块（2-3句话）
- [ ] 加粗 ≤ 6处
- [ ] 无AI虚词
- [ ] 无排比收束

### 搜一搜SEO
- [ ] `<title>` 含人物全名
- [ ] 前120字含人物全名 + 最关联身份
- [ ] 至少1个 `<h2>` 含人名
- [ ] 人物全名正文中 ≥ 5次
- [ ] 可搜索实体 ≥ 8个

### 格式
- [ ] 信息框 ≤ 2个，强调框 ≤ 1个
- [ ] 全部内联 style
- [ ] HTML 完整可打开
- [ ] 封面图颜色与其他文章不同

## 第7步：输出HTML并通过发布前检测

### 样式规范

| 元素 | 样式 |
|------|------|
| 容器 | max-width:677px, 居中 |
| 正文 `<p>` | 16px, #333, 行高1.8, justify |
| 标题 `<h1>` | 22px, bold, 居中, #1a1a1a |
| 署名 `<p>` | 13px, #999, 居中 |
| 小标题 `<h2>` | 18px, bold, #1a1a1a, border-bottom:2px solid 主题色 |
| `<strong>` | color:#1a1a1a | ≤ 6处 |
| 信息框 `<p>` | background-color:#f8f8f8, border-radius:8px, 显式 font-family |
| 强调框 `<p>` | background-color:#f0f7ff, border-left:4px solid #1890ff, 显式 font-family |

### 微信兼容性

- 信息框/强调框用 `<p>` 标签（不是 `<div>`）
- 必须用 `background-color` 不能用 `background` 简写
- 信息框/强调框必须显式写 `font-family`
- HTML 标签之间不要留空行
- 不要用 flex、table 布局，不要用 position 定位，不要用 gap 属性

### 排版差异化（每篇文章不要套同一个模板）

| 元素 | 怎么变 |
|------|--------|
| 主题色 | 每篇换：樱花人物 #d4636a，商业人物 #1890ff，暗黑风 #333 |
| h2 样式 | 同一篇内至少2种：彩底白字标签式、左侧竖线式、底部横线式、纯加粗 |
| 信息框/强调框 | 背景色跟主题色配套 |

读者连续看3篇不应该觉得"怎么排版都一样"。

### 封面图配色

**每篇文章颜色必须不同！**

| 话题类型 | 推荐色系 | 渐变示例 |
|---------|--------|----------|
| 夫妻/情感 | 玫瑰金/暖红 | #1a1a2e → #d4636a → #2d132c |

## 核心规律（16篇数据复盘）

1. **选题决定80%，写作只决定20%。** 一篇选题对的文章随手写完，大概率跑赢一篇选题错的文章反复打磨三天
2. **"告诉你不知道的事"永远比"帮你重新理解已知的事"传播力强**
3. **爆款标题 = 具体数字 + 反转/冲突。** 无一例外
4. **同一个人，标题差6倍**（肖战三篇：好标题1004阅读 vs 差标题168阅读）
5. **"从A到B"式标题是毒药**，数据已验证
6. **冷门人物的好故事 > 顶流的成绩单**（卢洋洋 > 肖战）
7. **搜一搜热度是底线。** 人物没热度，标题再好也没用。写之前先在微信搜一搜里搜人名
8. **阅读量 = 选题 × 标题。分享率 = 文章质量。两者可以完全不相关**

## 对比示范

### 差的（热搜评论文）

```
标题：肖战说压力没那么大：125天拍101天，他凭什么不焦虑
开头：4月19日，肖战在安吉尔品牌直播里做冰染画。
问题：所有信息来自热搜本身，全文是评论和感想，读完没有新信息
```

### 好的（人物故事文）

```
标题：卢洋洋在韩国当了9个月女团队长，然后一纸禁令全没了
开头：2017年3月的一天，卢洋洋和另外两个中国女孩被通知收拾行李，离开韩国。
好在：开头直接甩场景，有时间有人物有动作；全文有完整人生弧线；
      采访原话、韩媒报道、队友去向等独家信息；读完了解了一个完整的故事
```

**核心区别：热搜评论文是"有观点没信息"，人物故事文是"有信息没说教"。**

对比：
- 差："这本身就是一种进步" ← 说教
- 好："有点可惜。如果再回到以前，可能我会努力做到更好。" ← 让读者自己品
